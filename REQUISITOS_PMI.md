## 1. Descripción del Proyecto

El Sistema de Gestión Docente es una aplicación web institucional destinada al Departamento de Ingeniería. Centraliza y digitaliza el proceso cuatrimestral de novedades docentes (altas, bajas y cambios de dedicación/cargo), la reserva de aulas y laboratorios para exámenes, la gestión de datos del portal del docente, el seguimiento de tareas internas y la consulta de información mediante un chatbot con acceso rol-dependiente.

El sistema opera dos veces por año, en los períodos previos al inicio de cada cuatrimestre.

| ------ | --------------------------------------------------------------------------- |

## 3. Stakeholders

| ID     | Stakeholder                                         | Rol en el sistema                                                   |
| ------ | --------------------------------------------------- | ------------------------------------------------------------------- |
| STK-01 | Jefe de Cátedra                                     | Carga y envía novedades docentes de su cátedra                      |
| STK-02 | Coordinador de Carrera                              | Revisa y aprueba novedades dentro de su carrera asignada            |
| STK-03 | Secretaría Académica del Departamento de Ingeniería | Revisa novedades de todo el departamento; parametriza el sistema    |
| STK-04 | Decanato                                            | Aprobación final del proyecto docente; carga docentes especiales    |
| STK-05 | Administrativos                                     | Parametrización de configurables del sistema                        |
| STK-06 | Docente                                             | Actualiza datos personales, declara horas, consulta su situación    |
| STK-07 | Secretaría Académica Universitaria                  | Actor externo; su respuesta es registrada manualmente en el sistema |

---

## 4. Supuestos y Restricciones

| Tipo        | Descripción                                                                                                                                                     |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Restricción | El proceso de novedades se ejecuta únicamente antes del inicio de cada cuatrimestre (2 veces por año).                                                          |
| Restricción | Cada docente puede registrar un único tipo de novedad (alta, baja o cambio de dedicación/cargo) en una sola cátedra por cuatrimestre.                           |
| Restricción | Los docentes sin novedades se incluyen automáticamente en el Excel final con sus datos previos, sin requerir acción.                                            |
| Restricción | Las materias compartidas entre carreras tienen distintos códigos internos por carrera; un único Coordinador es designado responsable de su aprobación.          |
| Supuesto    | La autenticación se realiza mediante el Active Directory institucional (Microsoft SSO).                                                                         |
| Supuesto    | La Secretaría Académica Universitaria es un actor externo al sistema; su intervención se registra manualmente por el Decanato o la Secretaría del Departamento. |
| Pendiente   | El flujo exacto de objeciones de la Secretaría Académica Universitaria está pendiente de investigación.                                                         |
| Pendiente   | El flujo de segunda instancia de reclamos post-aprobación está pendiente de investigación.                                                                      |
